Internet of Things 物联网

Before reading this paper, I want to know what the future generation computer systems refers to.

In summary, this paper introduces a scene of future that every objects will be connected into the internet and clouds will connect with each other to share the data of all the hardware around us.

In the second part, it introduces the ubiquitous computing in the next decade. It introduces a novel integrated sensor-actuator-internet framework and it predicted that this framework would be the core technology of shaping a smart environment. As we moved from web to web2.0 to web3.0, the need for data-on-demand using sophisticated queries increases, so it is necessary to deploy a large-scale wireless sensor network infrastructure that includes data management and processing, actuation and analytics.

In the third part, it introduces some definitions, trends and elements. Internet of Things is realized in three paradigms, middleware, sensors, and knowledge. It had been forecasted that IoT would take 5-10 years for market adoption. The whole development of technologies are shown in figure 2 and figure 3. We can clearly see the sequence of emerging technologies from human augmentation to preictive analytics.

In the fourth part, it introduces several application domains which will be impacted by the emerging internet of things. They categorize the application into 4 domains: Personal and Home, Enterprize, Utilities, Mobile.

In the fifth part, it introduces a cloud centric internet of things. The aneka cloud computing platform is a .NET-based application development Platform-as-a-Service(PaaS), which can utilize storage and compute resources of both public and private clouds.

In the sixth part, it introduces IoT sensor data analytics SaaS using Aneka and Microsoft Azure. Microsoft Azure is a cloud platform, offered by Microsoft, includes four components, SQL Azure, AppFabric, Azure Martketplace. There are several advantages of combing them. Aneka can overcomes the interoperability issues by providing a framework, which enables the creation of adaptors for different cloud infrastructures. Another important features is SaaS to be updated by the developers dynamically. Management extensibility framework (MEF) provides a simple solution to this problem. The MEF can improves the flexibility, maintainability and testability of large applications.

In summary, this paper was published in 2013, but had depicted the lifestyle that we are not living. I think we haven't been such a intelligent lifestyle but we are approaching.
